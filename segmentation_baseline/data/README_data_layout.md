# Data Layout for Semantic Segmentation Baseline

## Directory structure

```
data/
  raw/                 # Input LAS/LAZ point clouds
    scene_001.las      # one file per scene or strip
    scene_002.laz
    ...

  tiles/               # Prepared tile blocks (generated by prepare_dataset.py)
    scene_001_tile_000.npy
    scene_001_tile_001.npy
    ...

  splits/              # Train/val/test split files
    train.txt          # one tile filename per line
    val.txt
    test.txt
```

## Label convention

Semantic labels must be stored as an integer per point:

| ID | Class       |
|----|-------------|
| 0  | unlabeled   |
| 1  | vine_row    |
| 2  | olive_tree  |
| 3  | other       |

Labels can be provided as:
1. A `classification` or `label` extra dimension inside the LAS file, or
2. A sidecar `.npy` file with the same stem name (e.g., `scene_001_labels.npy`)
   containing an int array of shape `(N,)` matching the point count.

If labels are absent, the preparation scripts will still work but will set all
labels to `0` (unlabeled). Training requires at least some labeled tiles.

## Tile .npy format

Each tile is saved as a numpy structured file containing:
- `points`: (N, 3) float64 — XYZ coordinates (local, zero-centred)
- `features`: (N, F) float32 — selected per-point features
- `labels`: (N,) int32 — semantic label per point (0 if unlabeled)
- `source_file`: str — original LAS filename this tile came from
- `tile_origin`: (2,) float64 — XY origin of this tile in the original CRS

## Creating labeled data

Recommended labeling tools:
- CloudCompare (free, cross-platform) — manual classification brush
- Semantic labeling via QGIS with 2D reference ortho overlay
- Custom script using cluster IDs from the existing pipeline as starting labels

Start with a small labeled subset (5-10 scenes) before committing to full labeling.
